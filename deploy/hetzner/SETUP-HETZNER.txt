Репо тузилиши: `dressai-api/` (Express, :3000), `getdressai/` (статик + dev/proxy),
`mobile/` (Expo). VPSда одатда **фақат API** ишлайди; вебни Vercel/Cloudflare Pages
да хостинг қилиш тавсия этилади (бир доменда HTML + API аралашмасин).

Men sizning Hetzner akkauntingizga kira olmayman. Quyidagilarni O‘ZINGIZ qilasiz:
1) Hetzner Cloud’da server yarating
2) SSH bilan ulaning
3) Skriptlarni serverga yuklab yoki qo‘lda nusxalab ishga tushiring

--------------------------------------------------------------------------------
1) Hetzner’da server
--------------------------------------------------------------------------------
- Image: Ubuntu 22.04 LTS
- Type: masalan CX22 (2 vCPU, 4 GB RAM)
- SSH key qo‘shing

--------------------------------------------------------------------------------
2) Birinchi marta (root sifatida)
--------------------------------------------------------------------------------
  scp -r deploy/hetzner root@SERVER_IP:/root/dressai-hetzner
  chmod +x /root/dressai-hetzner/scripts/*.sh
  /root/dressai-hetzner/scripts/01-bootstrap-root.sh

Bu: Node 20, git, ufw, Caddy, `dressai` user, repo klon (`/home/dressai/getdressai.com`),
    systemd `dressai.service` (ishchi каталог: `dressai-api/`).

--------------------------------------------------------------------------------
3) .env (dressai-api)
--------------------------------------------------------------------------------
  nano /home/dressai/getdressai.com/dressai-api/.env

`dressai-api/.env.example` bo‘yicha to‘ldiring (JWT, Supabase service role, Stripe, ...).

  chown dressai:dressai /home/dressai/getdressai.com/dressai-api/.env
  chmod 600 /home/dressai/getdressai.com/dressai-api/.env

--------------------------------------------------------------------------------
4) Caddy (HTTPS → API :3000)
--------------------------------------------------------------------------------
  nano /etc/caddy/Caddyfile

`Caddyfile.example`: `api.getdressai.com` → reverse_proxy 127.0.0.1:3000 (тавсия).
Агар бутун доменни API га берсангиз, статик веб шу доменда очилмайди.

  systemctl reload caddy

Cloudflare: SSL "Full (strict)"; origin — Hetzner IP.

--------------------------------------------------------------------------------
5) Bir marta build (npm ci)
--------------------------------------------------------------------------------
  sudo -u dressai /home/dressai/hetzner-deploy/02-npm-build.sh

--------------------------------------------------------------------------------
6) Keyingi yangilashlar
--------------------------------------------------------------------------------
  sudo bash /home/dressai/hetzner-deploy/03-deploy-update.sh

--------------------------------------------------------------------------------
7) Tekshiruv
--------------------------------------------------------------------------------
  systemctl status dressai
  curl -sS http://127.0.0.1:3000/health

Tashqaridan (domen API га уланган бўлса): https://api.getdressai.com/health

--------------------------------------------------------------------------------
8) Stripe / Paddle webhook
--------------------------------------------------------------------------------
Stripe: `https://<API-домен>/billing/webhook`
Paddle: `https://<API-домен>/billing/paddle-webhook`
